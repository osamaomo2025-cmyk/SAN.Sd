## ١. مقدمة وهيكل تسمية رموز التصميم | 1. Token Architecture & Naming Conventions

تم تصميم وتطوير نظام ترميز الهوية الرقمية الموحدة (Sovereign Design Tokens) كطبقة هندسية مستقلة تمثل المرجع الوحيد لكافة الأبعاد البصرية والتفاعلية لمنصة وزارة التجارة والصناعة وكافة المنصات الحكومية المستقبلية لجمهورية السودان.

### أ. هرمية الرموز (Token Hierarchy)
يتبع النظام هيكلية هرمية من ثلاثة مستويات رئيسية لضمان الاستدامة وسهولة التحديث:
1.  **الرموز الأساسية (Global/Reference Tokens):** وهي القيم الأولية الخام مثل درجات اللون الأخضر والأبعاد بالبكسل دون ربطها بوظيفة معينة (مثال: `COLOR_SCALES.sudanGreen.500 = "#007A33"`).
2.  **الرموز السيادية الدلالية (Sovereign Semantic Tokens):** وهي الرموز التي تعبر عن دور العنصر داخل الواجهة وترتبط مباشرة بالوضع الداكن أو النهاري (مثال: `primary = "#007A33"` في النهاري، و `focusRing = "#C5A059"`).
3.  **رموز المكونات (Component-Specific Tokens):** الأبعاد والخصائص الفريدة للمكونات التفاعلية كارتفاع حقول الإدخال وبطاقات التداول (مثال: `form.inputHeight = "3rem"`).

### ب. قواعد التسمية (Naming Conventions)
تلتزم جميع الرموز بصيغة برمجية موحدة تدعم معيار الكود النظيف وصديقة لـ Figma Token Transformer:
*   `category-type-item-state`
*   مثال: `color-semantic-primary-hover`

## ٢. تفاصيل الرموز الـ ١٨ المعتمدة | 2. The 18 Token Category Specifications

تم تفصيل كافة الرموز برمجياً في المجلد `/src/components/DesignTokens.ts`. وفيما يلي بيان طريقة الاستهلاك وتأثيرها البصري:

### ١. رموز الألوان (Color Tokens)
*   توزيع متناسق لدرجات نماء الوطن الأخضر (`sudanGreen` 50-950) والذهب السيادي المعدني للثروات الباطنية (`sudanGold` 50-950).
*   ألوان دلالية عالية التباين لمعالجة حالات الرخص والتحقق الثنائي والشكاوى الحكومية وحماية المستهلك.

### ٢. رموز الخطوط والموازين (Typography Tokens)
*   الخطوط الرسمية المعتمدة: `DIN Next Arabic` (للمحتوى العربي بنظام RTL) و `DIN Next` (للمحتوى الإنجليزي بنظام LTR).
*   موازين متناسقة تبدأ من العناوين الإعلانية العريضة `Display XXL` (3.5rem) نزولاً إلى السلاسل البرمجية وتفاصيل الجداول الصغرى (0.75rem).

### ٣. رموز الفراغات الممنهجة (Spacing Tokens)
*   تعتمد على شبكة الـ **8px Base Grid** لمنع تشويه الهوية والترتيب البصري.
*   تترواح الفراغات من الهوامش الدقيقة `px2` (2px) إلى الفراغات الهيكلية العملاقة `px200` (200px).

### ٤. رموز المنحنيات والأقطار (Corner Radius Tokens)
*   تجنب الحواف العشوائية عبر توفير تدرج هندسي يبدأ من `none` (0px) إلى زوايا البطاقات الاستثمارية `large` (16px) وأزرار المعاملات الموثقة `pill` (9999px).

### ٥. رموز الظلال والهيبة البصرية (Shadow Tokens)
*   ظلال إدارية مهيبة لتحديد هرمية العناصر (من الظلال البسيطة `soft` للبطاقات الإخبارية إلى الظلال العميقة `high` والمشعة بمؤشر ذكي لمساعد الاستثمار التفاعلي بالذكاء الاصطناعي).

### ٦. رموز الحدود والهياكل (Border Tokens)
*   مواصفات دقيقة لتأكيد سلامة الأشكال وعزلها برباط رسمي (`thin` للحدود العادية و `strong` لتأكيد التركيز النشط).

### ٧. رموز الحركات الدقيقة والتسارع (Motion Tokens)
*   دعم سرعات استجابة تضمن رشاقة المعالجات الرقمية (`fast` بـ 150ms لهوفر الأزرار، و `normal` للتنقل بين علامات التبويب، و `slow` لتأثيرات بوابات المعاملات).

### ٨. رموز الشفافية المعتمدة (Opacity Tokens)
*   مستويات دمج واضحة تدعم معايير التباين العالية أثناء التحميل وحالات الإيقاف المانع للاحتكار.

### ٩. رموز الطبقات الهرمية (Z-Index Tokens)
*   بنية طبقات صارمة تمنع تداخل النوافذ والتحذيرات السيادية العاجلة.

### ١٠. رموز التخطيط الأساسي (Layout Tokens)
*   تحديد قياسات النوافذ الموحدة، ارتفاع شريط التنقل الحكومي (72px)، وعرض اللوائح الاستثمارية الممتدة.

### ١١. رموز الاستجابة للشاشات (Responsive Breakpoints)
*   هندسة متجاوبة تضمن ثبات العرض على أجهزة الهواتف الصغيرة والحواسب الكبيرة وأجهزة الخدمات الرقمية الذاتية المنتشرة بالبلاد.

### ١٢. رموز شبكات العرض (Grid Tokens)
*   تقسيم الأعمدة للوصول لهياكل Bento Grid محكمة ومصانة بنظام الهوامش الجانبية (Margins).

### ١٣. رموز الأيقونات والرموز التعبيرية (Icon Tokens)
*   أحجام ثابتة للأيقونات الرسمية لضمان انسجام الهوية (من 16px لعلامات الاختيار الصغرى إلى 48px لأوسمة الختم الرقمي).

### ١٤. رموز الرسوم البيانية والمعلوماتية (Data Visualization Tokens)
*   ألوان متوافقة مع معايير الدمج لتمثيل مؤشرات كفاءة المصانع، صادرات الصمغ العربي، وإحصائيات الواردات والصادرات الاستراتيجية.

### ١٥. رموز النماذج والاستمارات (Form Tokens)
*   هندسة حقول المدخلات لتسهيل تعبئة المعاملات ووضوح مؤشرات الوجوب والاختيار.

### ١٦. رموز الجداول والبيانات الصارمة (Table Tokens)
*   خصائص عرض للبيانات الاستثمارية المعقدة مع ثبات الترويسة وسهولة العزل للسطر المحدد.

### ١٧. رموز الإشعارات والتحذيرات الحكومية (Notification Tokens)
*   تدرجات ألوان وتفاصيل تصميم مخصصة للتنبيهات العاجلة والطوارئ الاقتصادية القومية.

### ١٨. رموز الذكاء الاصطناعي التوجيهي (AI Tokens)
*   هوية بصرية ساحرة بتمثيل ذهبي وتدرجات أثيرية للمساعد الذكي لوزارة التجارة والصناعة، مع مؤشرات واضحة تعبر عن دقة وتوصيات الآلة الخاضعة للإشراف البشري.

## ٣. ربط وتكامل رموز التصميم مع Tailwind CSS و CSS Variables | 3. Installation Guide

### أ. تهيئة ملف `index.css` (CSS Custom Properties)
لتمكين التبادل الفوري بين الوضع الداكن والنهاري، يتم تحميل الرموز كمتغيرات CSS في جذر التطبيق:

```css
@import "tailwindcss";

:root {
  /* Spacing Variables */
  --space-2: 0.125rem;
  --space-8: 0.5rem;
  --space-16: 1rem;
  --space-24: 1.5rem;

  /* Typography Fonts */
  --font-arabic: "DIN Next Arabic", "Inter", sans-serif;
  --font-english: "DIN Next", "Inter", sans-serif;

  /* Corner Radius */
  --radius-xs: 4px;
  --radius-md: 12px;
  --radius-lg: 16px;

  /* Semantic Palette (Default Light Theme) */
  --background: #F4F6F5;
  --surface-card: #FFFFFF;
  --primary: #007A33;
  --primary-hover: #006E2E;
  --secondary: #C5A059;
  --text-primary: #1A1A1A;
  --border-thin: #E5E7EB;
}

[data-theme="dark"] {
  /* Semantic Palette (Dark Theme overrides) */
  --background: #0A0A0A;
  --surface-card: #141414;
  --primary: #007A33;
  --primary-hover: #009C41;
  --secondary: #C5A059;
  --text-primary: #F5F5F5;
  --border-thin: #262626;
}
```

### ب. تكامل المتغيرات مع `tailwind.config.js`
يتم تمرير مخرجات JSON البرمجية مباشرة إلى ملف الإعدادات لـ Tailwind CSS:

```javascript
module.exports = {
  theme: {
    extend: {
      colors: {
        sudan: {
          green: "var(--primary)",
          gold: "var(--secondary)",
          background: "var(--background)",
          card: "var(--surface-card)"
        }
      },
      fontFamily: {
        sans: ["var(--font-arabic)", "var(--font-english)", "sans-serif"]
      },
      borderRadius: {
        xs: "var(--radius-xs)",
        md: "var(--radius-md)",
        lg: "var(--radius-lg)"
      }
    }
  }
};
```

## ٤. دليل الاستخدام والممارسات البرمجية المثالية | 4. Implementation Rules

1.  **حظر القيم المرتجلة (Strict No-Inline Rule):** يُمنع منعاً باتاً كتابة الألوان أو الهوامش يدوياً داخل كود React. يجب دائماً استخدام صنف التنسيق المناسب المستمد من متغيرات الرموز (مثل استخدام `className="p-4 rounded-xl bg-sudan-green text-white font-bold"`).
2.  **استقرار الهوية (Theme Resilience):** عند التبديل إلى الوضع الداكن، اعتمد دائماً على الفئات المحددة بمفتاح الرموز الدلالي، لضمان استجابة بطاقات التسجيل بشكل مريح لعين المستخدم.
3.  **تبديل النصوص ثنائي الاتجاه (Bi-directional Logic):** يتم توظيف خصائص `dir="rtl"` أو `dir="ltr"` مع استخدام فئات محاذاة النص المنبثقة من النظام تلقائياً للحفاظ على هيبة ومقروئية الخطوط DIN Next Arabic في كلا اللغتين.
